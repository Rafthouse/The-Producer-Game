# 💿 Producer Tycoon

**Сатиричний симулятор музичного продюсера.**

Ви керуєте музичним лейблом. До вас у студію приходять дивні артисти різних
жанрів — від панків до поп-дів і бардів. У кожного свої характеристики,
особливості та абсурдний текст пісні. Ви вирішуєте: **підписати** чи
**відмовити**. Підписаний артист записує дебютний сингл — і тут починається
найцікавіше.

> «Менеджер втік із касою на електросамокаті
> Я перерізав стрічку біля сільського сортиру
> Овації стихли, бо то працював вентилятор
> Наше кохання згоріло разом із кашею на плиті»

---

## 🎮 Один повний ігровий цикл (MVP v0.1)

```
Новий артист → Підписання → Реліз → Результат → Новий артист
```

- **Екран 1 — Новий артист.** Картка з ім'ям, жанром, віком, 5 характеристиками,
  особливостями та текстом пісні. Кнопки `Підписати` / `Відмовити`
  (після відмови одразу генерується новий артист).
- **Екран 2 — Студія звукозапису.** Артист, назва треку та текст пісні.
  Кнопка `Випустити сингл`.
- **Екран 3 — Результат.** Тип успіху, прослуховування, нові фани, прибуток
  та 1–3 події після релізу. Кнопка `Новий артист`.

Статистика лейбла (баланс, фани, релізи) накопичується між циклами.

## ⚙️ Як це працює

- **Генерація артистів** — випадкові імена (готові + комбіновані), жанрові
  схильності характеристик, 1–3 риси з пулу.
- **Генерація текстів** — понад **120** абсурдних рядків, з яких збирається
  чотиривірш.
- **`calculateRelease()`** — бал залежить від таланту, харизми, репутації,
  залежності, рис та фактора удачі. Результат → один із 5 типів успіху:
  *Провал · Локальний мем · Нормальний реліз · Хіт · Культовий шедевр*.
- **Події** — підбираються залежно від успіху (хіти тягнуть добрі події,
  провали — погані; «дикі» можливі завжди й залежать від «хаосу» рис артиста).

Усі дані локальні. Без бекенду, без онлайну.

## 🛠️ Технології

- [React 18](https://react.dev/) + [TypeScript](https://www.typescriptlang.org/)
- [Vite](https://vite.dev/) — збірка та dev-сервер
- [Zustand](https://github.com/pmndrs/zustand) — стан гри
- [Tailwind CSS](https://tailwindcss.com/) — стилі
- [Framer Motion](https://www.framer.com/motion/) — анімації переходів

## 🚀 Запуск

Потрібен **Node.js 18+**.

```bash
# 1. Встановити залежності
npm install

# 2. Запустити дев-сервер
npm run dev
```

Відкрийте адресу, яку покаже Vite (зазвичай <http://localhost:5173>).

### Інші команди

```bash
npm run build      # production-збірка у dist/
npm run preview    # локальний перегляд зібраної версії
npm run typecheck  # перевірка типів без збірки
```

## 📁 Структура проєкту

```
src/
├── components/
│   ├── screens/        # ScreenNewArtist, ScreenRelease, ScreenResult
│   ├── ui/             # Button
│   ├── ArtistCard.tsx
│   ├── Layout.tsx      # шапка лейбла + фон студії
│   └── StatBar.tsx
├── data/               # genres, names, traits, lyrics, titles, events
├── lib/                # генерація артистів/текстів, calculateRelease, події
├── store/              # Zustand-стор гри
├── types/              # доменні типи (Artist, ReleaseResult, ...)
├── App.tsx             # маршрутизація екранів + AnimatePresence
├── main.tsx
└── index.css
```
